## Context

当前 `GET /market/market-diagnosis` 端点返回5项指标的综合诊断，判定"趋势市 vs 震荡市"。但它不回答"资金在往哪类资产流动"这个问题。系统已有申万行业指数数据（`index_daily`）、东财概念板块资金流（`em_sector_flow.py`）、以及8大类的概念分类体系（`prompt_seeds.py`），这些基础设施可以直接支撑风格轮动检测。

风格信号需要影响三层：QQ推送展示、AI交易提示词、仓位计算。这三层都已经在读 `market_diagnosis` 表，新增字段放在 `indicators_json` 中即可自然流通。

## Goals / Non-Goals

**Goals:**
- 在 `/market/market-diagnosis` 新增第6项指标 `style_rotation`，输出进攻/防御/资源避险/均衡四种模式
- 价格维度用 `index_classify` 动态获取 SW 行业代码，对比三个风格篮子的近5日涨跌幅
- 资金维度用东财概念板块主力净流入，按关键词聚合到三个风格篮子，对比近10日资金偏好
- 两维度双确认 + 连续3天跑赢触发模式切换，对手跑赢1天退出
- 风格信号注入 `trade_graph.py` AI 提示词和 `indicator.py` 仓位计算

**Non-Goals:**
- 不新增 Tushare API 权限依赖（`index_classify` 与现有 `index_daily` 用同一权限等级）
- 不改 `market_diagnosis` 表结构（`indicators_json` Text 字段直接容纳）
- 不改 `morning_diagnosis.py` 的执行流程，只在 `format_message()` 加展示
- 不引入新的外部数据源

## Decisions

### 1. SW行业代码：动态获取而非硬编码

**选择**: 用 `pro.index_classify(level='L1', src='SW2021')` 动态获取全部33个L1行业代码，按名称关键词映射到三个篮子。

**理由**: `index_classify` 与现有 `index_daily` 属同一 Tushare 接口族，不增加权限门槛。动态获取避免硬编码代码变更导致静默失败。`is_pub` 字段可过滤未发布指数。

**替代方案**: 硬编码16个代码 → 更简单但脆弱，Tushare 代码历史上有过变更。

### 2. 篮子映射：名称关键词匹配

```python
STYLE_BASKET_KEYWORDS = {
    "defense":  ["银行", "公用", "煤炭", "石油石化", "交通运输", "食品饮料"],
    "tech":     ["电子", "计算机", "传媒", "通信", "电力设备", "机械设备"],
    "resource": ["有色金属", "基础化工", "钢铁"],
}
```

`index_classify` 返回 `industry_name` 字段，用关键词子串匹配分入三个篮子。未匹配行业（如农林牧渔、房地产、美容护理等）不入任何篮子。

**理由**: SW行业名称是标准化的，不会像概念板块那样变化。匹配规则简单可靠。

### 3. 资金维度：东财概念板块 + 关键词聚合

**选择**: 保留概念板块聚合方案（原方案C），原因：
- `moneyflow_dc`（个股资金流）分批查询 200+ 只成分股 × 10 天的 API 调用量过大
- 东财概念板块 `main_net` 已经是聚合好的板块级资金流，数据质量好
- 9:10 AM 执行时东财降级到 Tushare `moneyflow_ind_dc`，同样可以按概念名称关键词聚合

**聚合方式**: 调用底层 `get_sector_flow("concept", top_n=300)` 获取全量概念板块，按科技/防御/资源关键词分类，求和每类的 `main_net` 和平均 `pct_change`。

**关键词去重**: 匹配时按优先级顺序（科技 → 资源 → 防御），每个概念只归属一个篮子，避免"AI芯片"被重复计数。

### 4. 状态机：三状态 + 双维度确认

```
          ┌─────────────────────────────────┐
          │           NEUTRAL (均衡)         │
          └───────────┬─────────────────────┘
                      │
          ┌───────────┼─────────────────────┐
          ▼           ▼                     ▼
    ┌──────────┐ ┌──────────┐ ┌──────────────────┐
    │ DEFENSE  │ │ OFFENSE  │ │ RESOURCE_HEDGE   │
    │ (防御)   │ │ (进攻)   │ │ (资源避险)        │
    └──────────┘ └──────────┘ └──────────────────┘
    
    进入: 价格+资金双维度同向跑赢 ≥ 3天
    退出: 对手跑赢 1天 → NEUTRAL（不会立刻反转，需重新累计3天）
```

**双确认规则**: 价格维度和资金维度必须同时指向同一方向，且各自连续跑赢天数都 ≥ 3，才触发模式切换。单维度信号不触发。

**背离检测**: 当价格和资金维度连续矛盾 ≥ 5 天，输出 `⚠️ 价格与资金背离` 警告，建议轻仓观望。

### 5. 每日跑赢判定

**价格维度**: `篮子A平均涨跌幅 > 篮子B平均涨跌幅` → A跑赢B。需要 A 同时跑赢其他两个篮子才算"A跑赢日"。

**资金维度**: 概念板块聚合后 `篮子A总主力净流入 > 篮子B总主力净流入` → A跑赢B。同样需要 A 同时跑赢其他两个篮子。

**三个篮子两两比较**: 每天判定 6 对比较结果（3×2），记录 `{winner}_win_days` 和 `{winner}_consecutive_days`。

### 6. API 调用优化

`index_classify` 结果缓存到模块级变量（SW行业分类不会每日变动），避免每次诊断都调一次。可用 `functools.lru_cache` 或手动缓存 + 每月刷新。

## Risks / Trade-offs

- **[风险] `index_classify` 返回代码与 `index_daily` 实际可用代码不一致** → 缓解：用 `is_pub='1'` 过滤 + 实际调用时 try/except 跳过失效代码
- **[风险] 概念板块关键词匹配可能边界不准** → 缓解：用优先级顺序避免重复归属；后续可人工维护关键词黑名单
- **[风险] 9:10 AM 执行时完全没有实时数据** → 缓解：这个指标用日频数据（Tushare 前一天 + 东财降级 Tushare），不依赖实时行情，9:10 执行恰好合适
- **[权衡] 成分股聚合 vs 概念板块聚合**：概念板块数据已有现成封装，开发量小；成分股聚合更精确但 API 调用量 ×10。当前阶段选概念板块，后续可升级
