## Context

当前交易 AI 的主线选择仅依赖单日概念板块数据（当日主力资金 TOP5 ∩ 当日涨幅 TOP5）。在震荡市中，单日数据容易被一日游板块诱杀——某些板块当天爆发但次日即回落，AI 追高被套。

需要增加 5 日持续性维度，形成"双轨"主线确认：
- **明线（Bright Track）**：现有逻辑，单日资金 TOP5 ∩ 单日涨幅 TOP5，捕捉当日爆发力
- **暗线（Dark Track）**：新增逻辑，5 日累计涨跌幅排名 + 5 日上涨天数排名，以当日资金为门控，捕捉持续性趋势

两条线平等权重，共同服务于最终选股决策。

数据来源：Tushare `moneyflow_ind_dc` API，支持按日期范围查询概念板块日频资金流向数据。

## Goals / Non-Goals

**Goals:**
- 后端新增 `GET /api/v1/market/concept-fund-flow-5d` 端点，返回暗线评分排行
- 前端工具 `get_concept_fund_flow_5d` 供 AI 调用
- 交易 Prompt（index.ts + prompt_seeds.py）替换为双轨主线选择逻辑
- 暗线评分模型：涨跌幅排名分 × 0.5 + 上涨天数排名分 × 0.5，当日主力净流入 > 0 为门控
- 三等优先级选股：双轨共振 > 暗线高分 ≥ 7 分 > 明线独有

**Non-Goals:**
- 不修改已有 API 端点
- 不涉及数据库迁移
- 不修改前端 UI 组件
- 不增加新的外部数据源（仅使用 Tushare）
- 不调整明线（现有单日逻辑）的打分或排序方式

## Decisions

### Decision 1: 使用 Tushare moneyflow_ind_dc 而非东方财富 push2

**选择**：Tushare `moneyflow_ind_dc` 查询 5 个交易日数据

**理由**：
- 东方财富 push2 仅提供实时/当日数据，无历史回溯能力
- Tushare 支持 `start_date`/`end_date` 范围查询，一次调用可获取全部 5 日数据
- 已有 `_get_tushare_pro()` 辅助函数和 `moneyflow_ind_dc` 使用经验（现有 `/concept-fund-flow` 已作为 fallback 使用）
- 返回字段完整：`pct_change`, `net_amount`, `trade_date`, `name` 等，满足聚合计算需求

**替代方案**：akshare `stock_sector_fund_flow_rank` — 仅提供单日数据，不适用。

### Decision 2: 5 个交易日窗口（非 5 个自然日）

**选择**：使用 Tushare `trade_cal` 过滤交易日，取最近 5 个交易日

**理由**：
- 周五到周一的 5 个自然日实际只有 2-3 个交易日
- 使用日历日会导致暗线数据量不足
- `trade_cal` 已是项目中已有的工具函数，直接复用

### Decision 3: 后端聚合而非前端聚合

**选择**：后端 `/concept-fund-flow-5d` 完成全部聚合计算，返回最终评分

**理由**：
- 5 日 × 400+ 概念 = 2000+ 条原始数据，不应传输到前端
- 聚合逻辑（SUM pct_change, COUNT pct>0, SUM net_amount, 排名打分）适合在后端处理
- AI 工具只接收最终评分排行，减少 token 消耗
- 与现有 `/concept-fund-flow` 模式一致：后端计算，AI 消费

### Decision 4: 评分模型

**选择**：`score = pct_rank_score × 0.5 + up_days_rank_score × 0.5`

排名分规则：
- 按 5 日累计涨跌幅降序排列，TOP10 得分 10→1（第 1 名 10 分，第 10 名 1 分），10 名以外 0 分
- 按 5 日上涨天数降序排列，TOP10 得分 10→1，10 名以外 0 分
- 满分 10 分（10 × 0.5 + 10 × 0.5）

**理由**：
- 涨跌幅和上涨天数各占 50%，兼顾"涨了多少"和"涨了多久"
- 排名分而非绝对值分：避免单日暴涨板块（如 +15% 一天）统治排名，5 天均匀上涨的板块更有优势
- TOP10 以外 0 分：减少噪音概念对 AI 判断的干扰

### Decision 5: 当日资金门控

**选择**：暗线候选必须满足 `今日主力净流入 > 0`，否则直接剔除

**理由**：
- 5 日累计好但今日出货 = 趋势可能反转，不应追入
- 门控是硬性过滤，不是评分因子——今日流出的板块不论历史多好都不进入暗线
- 与明线的"主力资金 TOP5"逻辑呼应，但暗线门控更宽松（> 0 即可，不要求 TOP5）

### Decision 6: 三等优先级选股

**选择**：

| 优先级 | 条件 | 仓位建议 |
|--------|------|----------|
| 第一等 | 板块同时在明线 TOP5 和暗线总分 ≥ 7 | 10-15% |
| 第二等 | 板块仅在暗线且总分 ≥ 7 | 8-12% |
| 第三等 | 板块仅在明线 | ≤ 3% |

**理由**：
- 双轨共振 = 既有当日爆发力又有 5 日持续性，确定性最高
- 暗线高分 = 慢牛板块（电力、红利等），虽不在一日榜单但持续上涨，可信度高于单日爆发
- 明线独有 = 一日游风险板块，严格控制仓位，仅做短线试探

### Decision 7: Prompt 两处维护

**选择**：同时修改 `servers/pi-server/src/index.ts` 和 `backend/app/db/prompt_seeds.py`

**理由**：
- `index.ts` 是 pi-server 运行时的 Prompt
- `prompt_seeds.py` 是数据库种子，用于初始化和重置
- 两处内容必须一致，否则数据库重置后行为回退
- 项目已有此模式（7 处重复定义），不在此变更中引入额外解耦

## Risks / Trade-offs

**Risk 1: Tushare API 频率限制**
- Tushare 免费用户有每分钟调用次数限制
- 缓解：暗线 API 和明线 API 共享同一 Tushare 连接，单次暗线调用只需 1 次 `moneyflow_ind_dc` 请求（带日期范围）

**Risk 2: 5 日数据缺口**
- 新上市概念板块可能不满 5 个交易日，累计涨幅失真
- 缓解：上涨天数 ≤ 2 的概念在排名时排到末尾，自然得分低

**Risk 3: Prompt 长度增加**
- 双轨逻辑比单轨逻辑复杂，Prompt 预计增加约 50 行
- 缓解：用简洁的结构化格式描述优先级表，避免冗长解释

**Risk 4: 暗线门控可能漏掉反转板块**
- 今日主力净流入 ≤ 0 但 5 日趋势极好的板块会被过滤
- 这符合设计意图：暗线找的是"正在持续且今天仍在流入"的板块，反转捕捉不属于暗线职责

**Risk 5: T+1 数据滞后**
- Tushare `moneyflow_ind_dc` 通常在收盘后更新，盘中调用可能无当日数据
- 缓解：盘中暗线使用前 4 个完整交易日 + 当日（若有），实际测试中 Tushare 数据通常在 15:30 后可查当日
- 备选：盘中可将今日数据标记为"未确认"降权，但 V1 不做此优化
